Overview

Two fundamentally different types of AI model are compared here. BERT is a discriminative model — built to classify. GPT-4 is a generative model — built to produce language, which can be directed toward classification through prompting.1

Understanding this distinction is essential before drawing any conclusions about which approach is better for a given business context.

Why This Comparison Matters

For a company processing thousands of customer reviews per day, the choice of model has real operational consequences:

Consideration BERT (fine-tuned) GPT-4 (prompting)
Setup cost High — requires labelled data & training Low — just write a prompt
Inference cost Low — runs locally High — API cost per call
Latency Fast (~10ms/sample) Slow (~1–3s/sample)
Accuracy High on in-domain data Strong zero-shot, improves with examples
Flexibility Retrain for new tasks Prompt engineering only

Evaluation Metric

The primary metric is F1-score, which balances precision and recall — important when both false positives and false negatives carry a cost:

\[F_1 = 2 \cdot \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}\]

ROC-AUC is used as a secondary metric:

\[\text{AUC} = \int_0^1 \text{TPR}(t) \, d\text{FPR}(t)\]

BERT Tokenisation and Dataset

Code 
MODEL_NAME = 'bert-base-uncased'
MAX_LEN    = 128

tokenizer = BertTokenizer.from_pretrained(MODEL_NAME)

class SentimentDataset(Dataset):
    def __init__(self, encodings, labels):
        self.encodings = encodings
        self.labels    = labels
    def __len__(self):
        return len(self.labels)
    def __getitem__(self, idx):
        item = {key: val[idx] for key, val in self.encodings.items()}
        item['labels'] = torch.tensor(self.labels[idx], dtype=torch.long)
        return item

train_encodings = tokenizer(train_texts, truncation=True,
    padding='max_length', max_length=MAX_LEN, return_tensors='pt')
val_encodings   = tokenizer(val_texts,   truncation=True,
    padding='max_length', max_length=MAX_LEN, return_tensors='pt')
test_encodings  = tokenizer(test_texts,  truncation=True,
    padding='max_length', max_length=MAX_LEN, return_tensors='pt')

train_dataset = SentimentDataset(train_encodings, train_labels)
val_dataset   = SentimentDataset(val_encodings,   val_labels)
test_dataset  = SentimentDataset(test_encodings,  test_labels)

Fine-Tuning BERT

Code 
model = BertForSequenceClassification.from_pretrained(
    MODEL_NAME, num_labels=2,
    id2label={0: 'NEGATIVE', 1: 'POSITIVE'},
    label2id={'NEGATIVE': 0, 'POSITIVE': 1}
).to(device)

def compute_metrics(eval_pred):
    logits, labels = eval_pred
    preds = np.argmax(logits, axis=-1)
    return {
        'accuracy' : round(accuracy_score(labels, preds), 4),
        'f1'       : round(f1_score(labels, preds, average='binary'), 4),
        'precision': round(precision_score(labels, preds, average='binary'), 4),
        'recall'   : round(recall_score(labels, preds, average='binary'), 4),
    }

training_args = TrainingArguments(
    output_dir='./bert_results',
    learning_rate=2e-5,
    per_device_train_batch_size=16,
    per_device_eval_batch_size=32,
    num_train_epochs=3,
    weight_decay=0.01,
    warmup_steps=500,
    eval_strategy='epoch',
    save_strategy='epoch',
    load_best_model_at_end=True,
    metric_for_best_model='f1',
    seed=42,
    fp16=torch.cuda.is_available(),
    report_to='none',
)

trainer = Trainer(
    model=model, args=training_args,
    train_dataset=train_dataset, eval_dataset=val_dataset,
    compute_metrics=compute_metrics,
    callbacks=[EarlyStoppingCallback(early_stopping_patience=2)]
)

train_result = trainer.train()
print(f"Final train loss: {train_result.metrics['train_loss']:.4f}")

GPT-4 Zero-Shot and Few-Shot Prompting

Code 
import openai
import time

client = openai.OpenAI(api_key="YOUR_API_KEY")

# Zero-shot prompt
ZERO_SHOT_TEMPLATE = """Classify the sentiment of this movie review as exactly one word: POSITIVE or NEGATIVE.

Review: {review}

Answer:"""

# Few-shot examples
FEW_SHOT_EXAMPLES = [
    {"review": "A visually stunning masterpiece.", "label": "POSITIVE"},
    {"review": "Heartfelt performances and a tightly written script.", "label": "POSITIVE"},
    {"review": "A feel-good crowd-pleaser.", "label": "POSITIVE"},
    {"review": "A tedious, poorly paced slog.", "label": "NEGATIVE"},
    {"review": "Confusing, loud, and devoid of emotional resonance.", "label": "NEGATIVE"},
    {"review": "Collapses under its own pretension.", "label": "NEGATIVE"},
]

def build_few_shot_prompt(review: str) -> str:
    examples = "\n".join(
        f'Review: {ex["review"]}\nSentiment: {ex["label"]}'
        for ex in FEW_SHOT_EXAMPLES
    )
    return (f"Classify each review as POSITIVE or NEGATIVE.\n\n"
            f"{examples}\n\nReview: {review[:400]}\nSentiment:")

def gpt4_predict(prompt: str) -> int:
    response = client.chat.completions.create(
        model="gpt-4",
        messages=[{"role": "user", "content": prompt}],
        max_tokens=5, temperature=0
    )
    reply = response.choices[0].message.content.strip().upper()
    return 1 if "POSITIVE" in reply else 0

Project Checklist

Key Takeaways

NoteWhat the comparison showed
  • BERT achieves higher F1 with low latency — the right tool for high-throughput production classification.
  • GPT-4 few-shot closes the gap significantly with just 6 labelled examples — no training required.
  • GPT-4 zero-shot performs surprisingly well, making it useful for rapid prototyping when labelled data doesn’t exist yet.
  • The practical decision depends on volume, latency constraints, and budget — not just accuracy numbers.
TipKey insight on few-shot prompting

Few-shot examples reduce label ambiguity — the model sees exactly what format and vocabulary are expected. It’s a zero-cost adaptation strategy: no GPU, no dataset labelling required.

Reflection

The most interesting tension in this project was between accuracy and practicality. BERT wins on pure metrics, but a business spinning up a sentiment analysis pipeline for the first time probably shouldn’t start with fine-tuning. GPT-4 with a well-written few-shot prompt gets you 85–90% of the way there in an afternoon.

The harder question — which I kept coming back to — is what “accuracy” even means in a production context. A model that’s 92% accurate on a balanced test set might perform very differently on the skewed, messy text that actually comes through a review system. Benchmark numbers are a starting point, not a verdict.

## Key Takeaways

::: {.callout-note}
## What the comparison showed

- **BERT** achieves higher F1 with low latency — the right tool for high-throughput production classification.
- **GPT-4 few-shot** closes the gap significantly with just 6 labelled examples — no training required.
- **GPT-4 zero-shot** performs surprisingly well, making it useful for rapid prototyping when labelled data doesn't exist yet.
- The practical decision depends on volume, latency constraints, and budget — not just accuracy numbers.
:::

::: {.callout-tip}
## Key insight on few-shot prompting
Few-shot examples reduce label ambiguity — the model sees exactly what format and vocabulary are expected. It's a zero-cost adaptation strategy: no GPU, no dataset labelling required.
:::

## Reflection

The most interesting tension in this project was between accuracy and practicality. BERT wins on pure metrics, but a business spinning up a sentiment analysis pipeline for the first time probably shouldn't start with fine-tuning. GPT-4 with a well-written few-shot prompt gets you 85–90% of the way there in an afternoon.

The harder question — which I kept coming back to — is what "accuracy" even means in a production context. A model that's 92% accurate on a balanced test set might perform very differently on the skewed, messy text that actually comes through a review system. Benchmark numbers are a starting point, not a verdict.
